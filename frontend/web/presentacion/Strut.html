Tema: 
Proceso de Prueba
ISO / IEC / IEEE 29119-2
 Objetivo:

Es definir un modelo de proceso genérico para las pruebas de
software que se puede utilizar dentro de cualquier ciclo de vida
del software de desarrollo.
El proceso de prueba se basa en un modelo de proceso de
tres capas que comprende:

  • Especificaciones de la prueba de organización (por ejemplo, 
la Política de prueba organizativa, estrategia de prueba de 
organización) 
  • Gestión de pruebas 
  • Las pruebas dinámicas
Un enfoque basado en el riesgo para las
pruebas se utiliza en toda la norma.
Las pruebas basadas en riesgo es un
enfoque de mejores prácticas para la
formulación de estrategias y la gestión
de las pruebas, ya que permite a las
pruebas que se priorizadas y se centró
en las características más importantes
y los atributos de calidad de cada sistema
bajo prueba.
